## 功能指南

### 1. 開始複習

**步驟：**
1. 進入自適應複習頁面
2. 點擊左側「開始複習」按鈕
3. 系統會：
   - 分析您的學習歷史
   - 運行 RL 模型進行推薦
   - 顯示 5 個推薦題目

**預期結果：**
```
✓ 頁面顯示推薦題目列表
✓ 每題顯示：
  - 技能名稱
  - 難度等級（簡單/中等/困難）
  - RL 分數
  - 推薦指數
```

### 2. 選擇題目

**步驟：**
1. 在推薦列表中點擊任意題目卡片
2. 卡片會高亮顯示（藍色邊框）
3. 反饋區會自動顯示

**視覺反饋：**
```
選中前：普通灰色卡片
選中後：藍色邊框 + 淡藍背景
```

### 3. 提交反饋

**可用選項：**
- **答對了** - 題目做對
- **答錯了** - 題目做錯
- **需要提示** - 請求提示（功能即將推出）

**步驟：**
1. 選擇題目
2. 點擊反饋按鈕
3. 系統會：
   - 更新您的學習歷史
   - 重新計算掌握度
   - 更新進度條
   - 加載新推薦

**進度更新：**
```
掌握度: 45% → 47%  (↑ 增加)
正確率: 75% → 77%  (↑ 增加)
進度條自動移動
```

### 4. 生成複習計畫

**步驟：**
1. 點擊左側「生成計畫」按鈕
2. 系統會分析並生成 3 場複習會話計畫
3. 顯示：
   - 初始掌握度
   - 目標掌握度
   - 預計提升幅度
   - 每場會話的目標

**計畫內容示例：**
```
計畫摘要：
初始掌握度: 45% → 目標掌握度: 80%
預計提升: 35% | 建議練習: 15 題

會話 1: [會話名稱]
  掌握度進度: 45% → 55%
  建議練習: 5 題 | 預期正確率: 70%

會話 2: [會話名稱]
  掌握度進度: 55% → 70%
  建議練習: 5 題 | 預期正確率: 75%

會話 3: [會話名稱]
  掌握度進度: 70% → 80%
  建議練習: 5 題 | 預期正確率: 80%
```

## 常見情況處理

### 情況 1: 推薦列表為空
```
可能原因：
1. 學習歷史不足
2. 系統初始化中

解決方案：
→ 點擊「開始複習」重新加載
→ 檢查是否已登入
```

### 情況 2: 頁面加載緩慢
```
可能原因：
1. RL 模型運行中
2. 網絡延遲

解決方案：
→ 等待加載完成（通常 3-5 秒）
→ 檢查網絡連接
→ 刷新頁面重試
```

### 情況 3: 數據未更新
```
可能原因：
1. API 連接失敗

解決方案：
→ 刷新頁面
→ 檢查瀏覽器控制台是否有錯誤
→ 重新登入系統
```

### 情況 4: 響應式顯示問題
```
移動設備上的調整：
✓ 側邊欄自動隱藏
✓ 布局變為單欄
✓ 按鈕自動調整大小
✓ 卡片全寬顯示
```

## 技術細節

### 系統架構
```
┌─────────────────────────────────────────┐
│      Flask Web 界面 (HTML/CSS/JS)      │
├─────────────────────────────────────────┤
│   RL 模型           AKT 模型            │
│ (推薦策略)      (知識追蹤)             │
├─────────────────────────────────────────┤
│      適應性複習 API (5 個端點)          │
├─────────────────────────────────────────┤
│          學習數據 & 題庫                 │
└─────────────────────────────────────────┘
```

### API 端點（已簡化版本調用）

| 端點 | 方法 | 功能 |
|-----|------|------|
| `/api/adaptive-review/start` | POST | 啟動複習會話 |
| `/api/adaptive-review/feedback` | POST | 提交答題反饋 |
| `/api/adaptive-review/plan/{student_id}` | GET | 生成複習計畫 |

## 🔧 技術細節

### 架構概述

```
┌─────────────────────────────────────┐
│         用戶界面層                   │
│  (adaptive_review_simple.html)       │
│  - 手寫板                            │
│  - 題目顯示                          │
│  - AI 對話                           │
│  - 統計信息                          │
└────────────┬────────────────────────┘
             │
             ▼
┌─────────────────────────────────────┐
│         API 層                       │
│  (adaptive_review_api.py)            │
│  - /start                            │
│  - /feedback                         │
│  - /question/<item_id>               │
│  - /analyze/<student_id>             │
│  - /health                           │
└────────────┬────────────────────────┘
             │
             ▼
┌─────────────────────────────────────┐
│         業務邏輯層                   │
│  (adaptive_review_mode.py)           │
│  - AKT 推論                          │
│  - RL 推薦                           │
│  - 弱項分析                          │
│  - 計畫生成                          │
└────────────┬────────────────────────┘
             │
             ▼
┌─────────────────────────────────────┐
│         數據層                       │
│  - 訓練數據                          │
│  - 模型檔案                          │
│  - 用戶歷史                          │
└─────────────────────────────────────┘
```

### 主要元件

#### 1. 前端 (HTML/CSS/JavaScript)
- **手寫板**: Canvas API 實現手寫功能
- **實時更新**: 動態 DOM 操作
- **API 通信**: Fetch API 調用後端

#### 2. 後端 (Flask + Python)
- **AKT 模型**: 知識追蹤推論
- **RL 模型**: 題目推薦決策
- **數據驗證**: 輸入格式檢查

#### 3. 數據流

```
用戶開始複習
    ↓
API: /start
    ↓
獲取推薦題目 (RL)
    ↓
顯示題目
    ↓
用戶提交答案
    ↓
API: /feedback
    ↓
更新 APR (AKT)
    ↓
推薦下一題
    ↓
迴圈...
```

## 🎓 使用範例

### 典型學生會話

```
1. 學生登入並訪問 /adaptive-review
2. 系統分析其學習歷史，計算 APR = 0.60
3. RL 模型推薦 5 個最優題目
4. 學生作答第一題，通過 API 提交
5. 系統更新 APR 為 0.63
6. AI 助手提供反饋和建議
7. 系統推薦下一題...
8. 經過 8 題練習後，APR 達到 0.75
9. 學生達到目標，會話結束
```

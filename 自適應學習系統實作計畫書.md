## **Phase 2: 後端能力更新演算法實作**

**目標**: 實作答題後「能力值更新」的核心計算邏輯。

**核心檔案**: `core/adaptive_config.py`, `core/adaptive_engine.py`, `core/routes/practice.py`

**詳細步驟**:

1.  **建立 `core/adaptive_config.py`**：
    ```python
    # core/adaptive_config.py
    ABILITY_DEFAULT, ABILITY_MAX = 20.0, 200.0
    CONCEPT_DEFAULT, CONCEPT_MAX = 20.0, 100.0
    CALCULATION_DEFAULT, CALCULATION_MAX = 20.0, 100.0
    AVG_ANSWER_TIME_SECONDS, MAX_ANSWER_TIME_SECONDS = 60, 180
    T_FACTOR_FAST, T_FACTOR_NORMAL, T_FACTOR_SLOW = 1.2, 1.0, 0.8
    ERROR_PENALTY_FACTOR = 0.6
    # ... (其他規則參數) ...
    ```

2.  **建立 `core/adaptive_engine.py`**：
    ```python
    # core/adaptive_engine.py
    from models import db, StudentAbility, TextbookExample
    from . import adaptive_config as config

    def update_student_ability(user_id, skill_id, question_id, is_correct, time_taken_seconds):
        question = db.session.get(TextbookExample, question_id)
        if not question: return
        H = question.difficulty_h

        ability = StudentAbility.query.filter_by(user_id=user_id, skill_id=skill_id).first()
        if not ability:
            ability = StudentAbility(user_id=user_id, skill_id=skill_id)
            db.session.add(ability)
        
        if is_correct:
            if time_taken_seconds < config.AVG_ANSWER_TIME_SECONDS: T_factor = config.T_FACTOR_FAST
            elif time_taken_seconds > config.MAX_ANSWER_TIME_SECONDS: T_factor = config.T_FACTOR_SLOW
            else: T_factor = config.T_FACTOR_NORMAL

            A_new = ability.ability_a + (H / 2) * (1 - ability.ability_a / config.ABILITY_MAX)
            U_new = ability.concept_u + (2 * (1 - ability.concept_u / config.CONCEPT_MAX) * T_factor) # 假設 I=1
            C_new = ability.calculation_c + (H * (1 - ability.calculation_c / config.CALCULATION_MAX) * T_factor)

            ability.ability_a = min(A_new, config.ABILITY_MAX)
            ability.concept_u = min(U_new, config.CONCEPT_MAX)
            ability.calculation_c = min(C_new, config.CALCULATION_MAX)
        
        db.session.commit()
    ```

3.  **修改 `core/routes/practice.py`**：
    ```python

    # ...

    def check_answer():

        if is_adaptive_mode:
            # ... 計算 time_taken ...
            update_student_ability(...)
        # ...
    ```

**預期成果**: 學生在自適應模式下答題後，後端會自動、準確地更新其個人能力值。

## **Phase 3: 題目推薦引擎實作**

**目標**: 建立一個能根據學生當前能力，從題庫中挑選出最合適題目的智慧推薦系統。

**核心檔案**: `core/adaptive_engine.py`, `core/routes/practice.py`

**詳細步驟**:

1.  **在 `core/adaptive_engine.py` 中建立 `recommend_question` 函式**:
    *   此函式將實現完整的 RS (Recommendation Score) 計算邏輯，包括所有5條規則的檢查與計分。
    *   它會遍歷題庫，為每道題計算一個 RS 分數，最終返回分數最高的題目物件。

2.  **在 `core/routes/practice.py` 中建立 `/get_adaptive_question` 路由**:
    ```python
    # core/routes/practice.py
    from core.adaptive_engine import recommend_question

    @practice_bp.route('/get_adaptive_question', methods=['GET'])
    @login_required
    def get_adaptive_question():
        # 1. 呼叫推薦引擎獲取最佳題目模板
        question_template = recommend_question(current_user.id, request.args.get('mode'))
        if not question_template:
            return jsonify({"error": "無合適題目"}), 404
            
        # 2. 使用 skill.generate() 動態生成具體題目
        mod = importlib.import_module(f"skills.{question_template.skill_id}")
        data = mod.generate(level=question_template.difficulty_level)

        # 3. 回傳包含 mode 和 question_id 的 JSON 給前端
        return jsonify({
            "question_id": question_template.id,
            "mode": "adaptive",
            "new_question_text": data["question_text"],
            "correct_answer": data["correct_answer"],
            # ...
        })
    ```

**預期成果**: 系統具備一個可用的 API 端點 (`/get_adaptive_question`)，能夠動態地為使用者推薦個人化的題目。
